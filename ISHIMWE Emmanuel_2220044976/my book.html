        3. Represents individuals who use the parking reservation system.

      3. Parkinglot Table

        4. Represents the overall parkingslot area or garage where parking spaces are located. Contains details about the facility, including its name, address, operating hours, and capacity.


      4. parkingslot Table

        5. Represents a physical parking spot within a parkinglot. it include:id, slot_number, size, status


      5. registration Table

      6. car Table

        7. the table that show the full information of parked vehicles in the slots.
        it contains :name, age,model,manufacturer.


      7. reservation Table

      Represents a booking made by a user to reserve a parking space.
      Contains information such as the user making the reservation, the parking space being reserved, start and end times, and any associated fees. . It has the following attributes: reservation_id, start_time, end_time, cost
